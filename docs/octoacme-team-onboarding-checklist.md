# OctoAcme — Team Onboarding Checklist

## Purpose
Help new team members quickly understand their role, collaboration patterns, and key resources needed for successful project participation.

## General Onboarding (All Roles)
- [ ] Review [Project Management Overview](octoacme-project-management-overview.md) to understand OctoAcme's approach
- [ ] Read [Roles and Personas](octoacme-roles-and-personas.md) to understand all team roles and interactions
- [ ] Access project repository and review README
- [ ] Join team communication channels (Slack, Teams, etc.)
- [ ] Attend team introduction meeting
- [ ] Review current sprint/iteration goals and backlog
- [ ] Understand the Definition of Done (DoD) for the project

## Role-Specific Onboarding

### Developer
- [ ] Set up local development environment
- [ ] Review coding standards and PR conventions
- [ ] Understand CI/CD pipeline and automated testing
- [ ] Connect with DevOps Engineer for infrastructure questions
- [ ] Review architecture documentation
- [ ] Pair with existing team member for first task

### Product Manager
- [ ] Review product vision and roadmap
- [ ] Understand success metrics and KPIs
- [ ] Meet with key stakeholders
- [ ] Access analytics and user research data
- [ ] Align with UX Designer on current user needs
- [ ] Review backlog prioritization criteria

### Project Manager
- [ ] Review project charter and timeline
- [ ] Understand risk register and dependencies
- [ ] Set up status reporting cadence
- [ ] Connect with all core team members
- [ ] Review communication plan with stakeholders
- [ ] Understand escalation pathways

### Scrum Master
- [ ] Understand team's current agile maturity
- [ ] Review ceremony schedule (standups, planning, retros)
- [ ] Connect with Project Manager on coordination
- [ ] Understand team velocity and capacity
- [ ] Review impediment log and escalation process
- [ ] Set up metrics tracking (burndown, velocity)

### UX Designer
- [ ] Review existing design system and components
- [ ] Understand user research findings and personas
- [ ] Connect with Product Manager on product vision
- [ ] Set up design review cadence with Developers
- [ ] Access design and prototyping tools
- [ ] Review accessibility requirements

### QA/Testing
- [ ] Understand test strategy and coverage goals
- [ ] Review test environments and access
- [ ] Set up test automation framework
- [ ] Coordinate with DevOps Engineer on test infrastructure
- [ ] Review acceptance criteria standards
- [ ] Understand release validation process

### Technical Writer
- [ ] Review existing documentation structure
- [ ] Understand documentation standards and style guide
- [ ] Connect with Product Manager on documentation priorities
- [ ] Set up access to documentation tools and repositories
- [ ] Review release notes process and timeline
- [ ] Coordinate with Developers on technical content

### DevOps Engineer
- [ ] Review infrastructure architecture and cloud resources
- [ ] Understand CI/CD pipeline configuration
- [ ] Access monitoring and alerting systems
- [ ] Review incident response and on-call procedures
- [ ] Understand deployment windows and release process
- [ ] Coordinate with Developers on build requirements

## Collaboration Touchpoints

### Cross-Role Meetings to Expect
- **Daily Standups**: Developers, QA/Testing, Scrum Master
- **Sprint Planning**: Scrum Master, Developers, QA/Testing, Product Manager, UX Designer
- **Sprint Review/Demo**: All roles invited
- **Retrospectives**: All delivery team members (facilitated by Scrum Master)
- **Weekly Sync**: Project Manager, Product Manager, key stakeholders
- **Design Reviews**: UX Designer, Developers, Product Manager
- **Release Planning**: Project Manager, DevOps Engineer, QA/Testing, Technical Writer

### Key Handoff Points
- **Requirements → Design**: Product Manager to UX Designer
- **Design → Development**: UX Designer to Developers
- **Development → QA**: Developers to QA/Testing
- **QA → Deployment**: QA/Testing to DevOps Engineer
- **Feature Complete → Documentation**: Developers to Technical Writer
- **Pre-Release → Release**: DevOps Engineer coordinates with all roles

## First Week Goals
- [ ] Complete general and role-specific onboarding checklist
- [ ] Understand your primary collaboration partners
- [ ] Attend all relevant ceremonies and meetings
- [ ] Complete at least one meaningful contribution (task, review, or document)
- [ ] Identify any gaps in documentation or process and share feedback

## Questions or Issues?
- **Process questions**: Contact Project Manager or Scrum Master
- **Product questions**: Contact Product Manager
- **Technical questions**: Contact Lead Developer or DevOps Engineer
- **Design questions**: Contact UX Designer
- **Documentation questions**: Contact Technical Writer
